# Day 003 - Git Fundamentals

## What I did Today

- Created a README.md file for the Cloud Engineer Academy.
- Added project goals to the README.
- Added a study schedule to the README.
- Practiced using git status.
- Practiced using git diff
- Practiced using git diff --staged.
- Practiced using git restore.
- Created a .gitignore file.
- Learned how to ignore log files, temporary files, and environment files.
- Updated the Handboog with Git fundamentals.
- Created a progress tracker to keep the academy organized.

## What I Learned

- git status shows the current state of the repository.
- git diff shows changes before staging.
- git diff --staged shows changes after staging but before committing.
- git restore can discard unwanted local changes.
- .gitignore prevents Git from tracking files that should not be committed.
- README.md explains what the project is about.
- I should review changes before committing.
- A progress tracker helps keep a long-term learning journey organized.

## Mistakes I Made

- I accidentally modified the README with a bad test line.
- I accidentally added extra characters to the study schedule.
- I had to use git restore and git diff to inspect and fix changes.
- I noticed that Day 2 was already recorded inside day-001.md, so we created a tracker to avoid losing context.

## What I Learned From Those Mistakes

I learned that Git helps me catch mistakes before they become permanent commits. I should never commit blindly. I should always use git status and git diff before saving changes.

I also learned that long-term projects need a source of truth. The progress tracker will help me keep track of what I already completed and what comes next.

## Important Lesson

A clean Git workflow is:

1. Edit files.
2. Run git status.
3. Run git diff.
4. Fix mistakes.
5. Stage changes with git add.
6. Review staged changes with git diff --staged.
7. Commit with a clear message.
