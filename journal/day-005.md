# Day 005 - GitHub workflow and Sync

## What I Did Today

- Practiced using git pull before working
- Confirmed my local repository was up to date with origin/main
- Updated README.md with Day 4 and Day 5 progress
- Reviewed changes with git diff
- Staged changes with git add
- Committed the README update
- Pushed the commit to GitHub
- Confirmed local main and origin/main were synchronized

### What I Learn

- git pull checks GitHub for new commits and brings them into my local repo
- git push uploads my local commits to GitHub
- origin/main represents the main branch on GitHub
- If Git says my branch is up to date with origin/main, local and remote are synced
- I should pull before working and push after comitting

## Mistakes I Made

- No major mistakes today
- I practiced checking status carefully before and after pushing

## Important Lesson

My normal workflow should be:

1. Start in the repo
2. Run git status
3. Run git pull
4. Make changes
5. Run git status
6. Run git diff
7. Stage changes
8. Run git diff --staged
9. Commit
10. Push
11. Confirm git status is clean
