## GitHub Connection

### Local Repository
A local repository is the Git project stored on my computer.

Example:

/home/juan/cloud-engineer-academy

### Remote Repository
A remote repository is the version of my project stored online, such as on GitHub.

Example:

github.com/JuanCamilo012599/cloud-engineer-academy

### origin
origin is the default name Git uses for the remote repository.

### git remote -v
Shows the remote repositories connected to my local project.

Example:

git remote -v

### git remote add origin
Connects a local repository to a remote repository for the first time.

Example:

git remote add origin git@github.com:username/repo.git

### git remote set-url origin
Changes the URL of an existing remote.

I used this when my origin was set to HTTPS and I wanted to switch it to SSH.

Example:

git remote set-url origin git@github.com:JuanCamilo012599/cloud-engineer-academy.git

### SSH Authentication
SSH allows my computer to authenticate with GitHub using a key pair.

Private key:

id_ed25519

Public key:

id_ed25519.pub

Important:

The private key should never be shared.
The public key is safe to add to GitHub.

### ssh -T git@github.com
Tests if my SSH connection to GitHub works.

### git push -u origin main
Pushes my local main branch to GitHub and sets it to track origin/main.

### origin/main
origin/main represents the main branch that lives on GitHub.

## GitHub Lesson

A professional Git workflow is:

1. Make changes locally.
2. Review changes with git status and git diff.
3. Commit changes locally.
4. Push commits to GitHub.

After the first push, I can usually use:

git push

# Day 004 - GitHub Connection

## What I Did Today

- Created a GitHub repository for cloud-engineer-academy.
- Learned the difference between a local repository and a remote repository.
- Checked that no remote was connected using git remote -v.
- Created an SSH key inside WSL Ubuntu.
- Added the public SSH key to GitHub.
- Tested the SSH connection with ssh -T git@github.com.
- Changed the Git remote from HTTPS to SSH.
- Pushed the local main branch to GitHub.
- Connected local main to origin/main.

## What I Learned

- A local repository lives on my computer.
- A remote repository lives online on GitHub.
- origin is the default name for the remote repository.
- SSH is a secure way to connect my machine to GitHub.
- The private SSH key should never be shared.
- The public SSH key can be added to GitHub.
- git push sends my local commits to GitHub.
- git push -u origin main connects my local main branch to GitHub’s main branch.

## Mistakes I Made

- I almost created a second local repository using GitHub Desktop.
- I first had origin configured with HTTPS.
- I tried to add origin again, but Git said remote origin already exists.
- I used the wrong SSH URL format with my email instead of my GitHub username.

## What I Learned From Those Mistakes

I learned that I should not create duplicate local repositories. My real project lives in WSL at:

/home/juan/cloud-engineer-academy

I also learned that SSH remote URLs use the GitHub username, not the email address.

Correct format:

git@github.com:GitHubUsername/repository.git

## Important Lesson

My GitHub workflow is now:

1. Work locally in WSL.
2. Check status with git status.
3. Review changes with git diff.
4. Stage files with git add.
5. Commit with git commit -m.
6. Push to GitHub with git push.
