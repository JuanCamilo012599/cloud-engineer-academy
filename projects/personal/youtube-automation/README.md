# YouTube Automation — Learning-in-Public Channel

## Status

Idea — not started.

## Problem Statement

Turn my Cloud Engineer Academy progress into a faceless, largely automated YouTube channel
that teaches Linux/Git/cloud fundamentals to other beginners, using the same
journal/handbook notes I'm already producing as source material.

## Open Questions

- Content pipeline: script generation from journal entries, text-to-speech, slides/visuals,
  video assembly, thumbnail generation, upload — which pieces are automated vs. manual first?
- What does "faceless" mean here exactly: voiceover-only with slides? Screen recordings of the
  terminal? AI narration?
- Publishing cadence relative to study cadence (2-3 hrs/day, Mon-Sat).

## Roadmap Connections

This is a strong real-world vehicle for Phase 2 (Python automation, APIs), Phase 5/6/7 (if the
pipeline runs on cloud infra, containers, or IaC instead of a local script), and Phase 11
(portfolio project — it's automation + a real shipped product).

## Next Steps

- Define the minimum viable pipeline (even fully manual for video #1) before automating anything.
- Decide the content format for episode 1.
